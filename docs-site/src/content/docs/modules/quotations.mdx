The Quotations module manages client quotations for Video Access Security Solutions Ltd projects. Quotations are organised under **Projects** — a project represents a client engagement and can have one or more quotations.

Open with **Cmd+2** (macOS) / **Ctrl+2** (Windows).

## Creating a project

Projects must be created before quotations. A project holds the client details that appear on printed quotes.

<Steps>
1. Click **+ New Project** or press the **+ New Project** button in the toolbar.
2. Fill in the project form:

   <table>
     <thead><tr><th>Field</th><th>Required</th><th>Notes</th></tr></thead>
     <tbody>
       <tr><td>Project Name</td><td>Yes</td><td>Descriptive name (e.g. "Accra Mall Phase 2")</td></tr>
       <tr><td>Client Name</td><td>Yes</td><td>The client organisation or individual</td></tr>
       <tr><td>Client Address</td><td>No</td><td>Physical or mailing address</td></tr>
       <tr><td>Attn</td><td>No</td><td>Attention — contact person at the client</td></tr>
       <tr><td>Tel</td><td>No</td><td>Client telephone number</td></tr>
     </tbody>
   </table>

3. Save. The project appears in the list.
</Steps>

## Creating a quotation

<Steps>
1. Press **Cmd+N** / **Ctrl+N** or click **+ New Quotation**.
2. Fill in the quotation form:

   <table>
     <thead><tr><th>Field</th><th>Required</th><th>Notes</th></tr></thead>
     <tbody>
       <tr><td>Reference</td><td>—</td><td>Auto-generated as <code>VASSL-YYYY-NNNN</code> — read-only</td></tr>
       <tr><td>Project</td><td>Yes</td><td>Select from the project dropdown</td></tr>
       <tr><td>Exchange Rate</td><td>No</td><td>JMD/USD rate — defaults to the rate set in Settings → Price Book</td></tr>
       <tr><td>Discount %</td><td>No</td><td>Discount applied to the subtotal (default 0)</td></tr>
       <tr><td>GCT %</td><td>No</td><td>Jamaica's General Consumption Tax (default 15 %)</td></tr>
       <tr><td>Validity</td><td>No</td><td>Days until the quote expires (default 30)</td></tr>
       <tr><td>Notes</td><td>No</td><td>Free-form text shown on the quote document</td></tr>
     </tbody>
   </table>

3. Save. The quotation opens in **Draft** status.
</Steps>

<Aside type="note">
Reference numbers are assigned automatically in the format `VASSL-YYYY-NNNN` (e.g. `VASSL-2026-0001`). They cannot be changed after creation.
</Aside>

<Aside type="tip">
The exchange rate field pre-fills with the current rate from **Settings → Price Book**. You only need to override it if this particular quotation uses a different rate. Keep Settings up to date and most quotations will not need manual rate entry.
</Aside>

## Adding line items

<Steps>
1. Open a quotation in **Draft** status — switch to the **Items** tab.
2. Click **+ Add Item** (enabled when a quotation is selected).
3. Optionally pick a product from the product picker — this auto-fills the **Description** and **Unit Price** from the most recent Price Book entry.
4. Review and adjust the line item fields:

   <table>
     <thead><tr><th>Field</th><th>Required</th><th>Notes</th></tr></thead>
     <tbody>
       <tr><td>Description</td><td>Yes</td><td>What this line covers</td></tr>
       <tr><td>Quantity</td><td>Yes</td><td>Must be greater than 0</td></tr>
       <tr><td>Unit</td><td>No</td><td>e.g. <code>ea</code>, <code>lot</code>, <code>set</code></td></tr>
       <tr><td>Unit Price (USD)</td><td>Yes</td><td>Price per unit — must be ≥ 0</td></tr>
       <tr><td>Discount %</td><td>No</td><td>Per-line discount, clamped 0–100 (default 0)</td></tr>
       <tr><td>Total</td><td>—</td><td>Computed read-only: Qty × Unit Price</td></tr>
     </tbody>
   </table>

5. Save.
</Steps>

<Aside type="note">
Line item prices are pulled from the **most recent Price Book entry** at the time you add the item. If prices change later, the quotation retains the price recorded at build time.
</Aside>

## Editing a quotation

You can edit the financial details of a quotation while it is in **Draft** status. Once a quotation has been sent, approved, or rejected, it can no longer be edited.

<Steps>
1. Select a **Draft** quotation in the list.
2. Click **Edit Quotation** in the detail sidebar.
3. The edit form opens pre-filled with the quotation's current values. Adjust the project, exchange rate, discount, GCT, validity, or notes as needed.
4. Click **Save Changes**.
</Steps>

<Aside type="note">
Editing a quotation does not change its line items. To change what is on the quotation, add, edit, or remove individual line items from the **Items** tab while the quotation is still in Draft status.
</Aside>

## Totals and JMD conversion

The quotation detail view shows a summary footer with:

- **Subtotal** — sum of all line items after per-line discounts
- **Discount** — the overall discount percentage applied to the subtotal
- **GCT** — General Consumption Tax applied after the discount
- **Total (USD)** — the final amount in US dollars
- **Total (JMD)** — the USD total converted at the quotation's exchange rate

The exchange rate used for the JMD total is the rate that was recorded on the quotation when it was created or last edited. If no rate was stored, VASSL falls back to the current rate in **Settings → Price Book**.

## Approving a quotation

When a client accepts a quotation:

<Steps>
1. Open the quotation.
2. Change the status to **Approved**.
3. VASSL automatically creates **stock entries** in Inventory for each line item, recording the outgoing stock against the project.
</Steps>

<Aside type="tip">
Approving a quotation is the primary mechanism for recording outgoing stock. You do not need to manually adjust inventory after a project is confirmed.
</Aside>

## Workflow summary

```
New Project → New Quotation (Draft)
                    │
                    ├─ Add / Edit Line Items
                    ├─ Edit Quotation (Draft only)
                    ├─ Mark as Sent
                    │
                    ├─ Approved ──► Stock entries created in Inventory
                    └─ Rejected
```
